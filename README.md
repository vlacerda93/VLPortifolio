Um portfólio web moderno desenvolvido para exibir minha trajetória, habilidades e projetos nas áreas de **Desenvolvimento de Software, DevOps e Cibersegurança**.

## 🌟 Funcionalidades do Projeto

- **Design Moderno:** Uso intensivo de Glassmorphism, garantindo um visual translúcido e futurista.
- **Efeitos Dinâmicos:** Animações sutis de background (como a bandeira de Pernambuco animada) e um efeito digital estilo "Matrix" (`matrix.js`) interativo nas seções principais.
- **Seção de Stacks Interativa:** Habilidades categorizadas (Web, DevOps & Segurança, Bancos de Dados) em formato *Accordion* para uma leitura mais limpa e organizada.
- **Showcase de Projetos:** Galeria responsiva (com Flexbox e CSS Grid) contendo capturas de tela e links diretos para projetos de mercado e open-source (ex: Fuinha Network Monitor, Escuna Mero e PrintBord).

## 🎓 Contexto Acadêmico (Aponti Academy)

Este repositório foi estruturado e desenvolvido como parte de um desafio focado em **Versionamento Colaborativo** e simulação de um ambiente real de mercado. 

### Metodologia de Trabalho: GitFlow
Para garantir a estabilidade do código, organização e evitar conflitos diretos, adotamos rigorosamente o modelo **GitFlow**, com papéis de ramificação específicos:
- `main`: Ambiente de produção impecável e intocável diretamente.
- `develop`: Ramificação centralizadora onde as integrações e testes ocorrem.
- `feature/*`: Todo e qualquer novo desenvolvimento de seção, conteúdo ou refatoração visual foi desenvolvido de maneira isolada em ramificações derivadas de `develop`.

### Code Review e Pull Requests
Nesta dinâmica, modificações não são simplesmente submetidas (*push*) para as ramificações principais. O processo exigiu:
1. O desenvolvimento de funcionalidades em ramos `feature/`.
2. Abertura de um **Pull Request (PR)** no GitHub.
3. A designação de um colega (dupla) como Revisor (Reviewer) oficial.
4. Análise estrita de código pelo revisor antes de conceder o *Approve* e finalizar o *merge* para a `develop`.
